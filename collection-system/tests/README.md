# Auth模块测试文档

## 1. 测试环境要求

- Node.js 14.x 或更高版本
- npm 或 yarn 包管理器

## 2. 安装测试依赖

在项目根目录下执行以下命令安装测试依赖：

```bash
npm install --save-dev jest supertest mock-fs
```

└── tests/

## 5. 运行测试

### 5.1 运行所有测试

在项目根目录下执行以下命令运行所有测试：

```bash
npm test
```

### 5.2 运行特定测试文件

运行工具函数测试：

```bash
npm test tests/auth/utils.test.js
```

运行API端点测试：

```bash
npm test tests/auth/api.test.js
```

### 5.3 运行特定测试用例

```bash
npm test tests/auth/api.test.js -t "用户登录 - 正确的凭据"
```

### 5.4 查看测试覆盖率

```bash
npm test -- --coverage
```

## 6. 测试注意事项

1. **测试隔离**：每个测试用例运行前都会重置模拟数据库，确保测试之间的独立性
2. **模拟数据**：使用mock-fs库模拟文件系统，避免测试影响真实数据
3. **环境变量**：测试环境使用与生产环境相同的密钥（`collection_system_secret_key`）
4. **API状态码**：测试中验证了各种API状态码，包括200、400、401、403、404等
5. **权限控制**：全面测试了管理员和普通用户的权限差异

## 7. 测试结果分析

测试运行完成后，会显示测试通过率和详细的测试结果。如果有测试失败，会显示失败原因和位置，便于调试和修复问题。

## 9. 常见问题及解决方案

### 9.1 测试失败：数据库路径错误

**问题**：测试中出现"ENOENT, no such file or directory"错误

**解决方案**：确保测试文件中模拟的数据库路径与server.js中定义的路径一致

### 9.2 测试失败：依赖库版本问题

**问题**：测试框架或依赖库版本不兼容

**解决方案**：使用package.json中指定的依赖版本，或更新依赖库到兼容版本

### 9.3 测试失败：API响应结构变化

**问题**：API响应结构发生变化导致测试失败

**解决方案**：更新测试用例中的响应结构验证，确保与实际API响应一致
